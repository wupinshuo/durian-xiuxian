# 功能规格目录

本目录包含榴莲修仙项目的功能规格文档，用于规划和设计新功能。

## 规格文档结构

每个功能规格应包含以下文件：

- `requirements.md`: 需求文档，描述功能的用户故事和验收标准
- `design.md`: 设计文档，描述功能的技术设计和架构
- `tasks.md`: 任务列表，描述实现功能的具体步骤

## 创建新规格

要创建新的功能规格，请按照以下步骤操作：

1. 在本目录下创建一个新的子目录，使用kebab-case命名（如：`new-feature`）
2. 在子目录中创建上述三个文件
3. 按照模板填写文档内容
